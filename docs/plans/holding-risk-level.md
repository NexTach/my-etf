# 종목 위험도 안내 기능 구현 명세

## 목표

운영 포트폴리오의 각 종목에 관리자가 정성적으로 지정하는 안내용 위험도를 저장하고, 공개 포트폴리오와 종목 상세에서 색상 배지로 표시한다.

## 확정 요구사항

- 위험도 상태는 `미지정`, `저위험`, `고위험`이다.
- 기존 종목의 초기 상태는 `미지정`이다.
- `저위험`은 파란색, `고위험`은 빨간색 배지로 표시한다.
- 색상뿐 아니라 `저위험`/`고위험` 문구를 함께 표시한다.
- 미지정 상태는 공개 화면에 표시하지 않는다.
- 관리자는 운영 포트폴리오 목록에서 현재 상태를 확인하고 종목 수정 모달에서 변경한다.
- 공개 화면은 메인 포트폴리오 종목 목록과 종목 상세에 표시한다.
- 위험도는 투자, 출금, 배당, 비중 산정 등 어떤 계산에도 사용하지 않는 안내용 메타데이터다.
- 변경 이력과 별도 공시는 남기지 않는다.

## 데이터 설계

- `PortfolioHolding`에 nullable `riskLevel` 필드를 추가한다.
- 저장값은 `LOW` 또는 `HIGH`이며 `NULL`은 미지정을 의미한다.
- 기존 행은 데이터 갱신 없이 `NULL` 상태를 유지한다.
- 값의 종류가 적고 조회 조건으로 사용하지 않으므로 인덱스는 추가하지 않는다.

## 서버 및 관리자 UI

- 보유 종목 조회 결과의 `Holding` 타입과 매핑에 위험도를 포함한다.
- 종목 등록/수정 API는 빈 값을 미지정으로, `LOW`/`HIGH`를 지정값으로 검증한다.
- 종목 upsert 시 미지정은 `NULL`로 저장한다.
- 관리자 목록에는 `미지정`을 포함한 현재 위험도를 표시한다.
- 수정 모달에는 `미지정 / 저위험 / 고위험` 선택 필드를 추가한다.

## 공개 UI 및 문서

- 메인 포트폴리오 종목명 옆에 지정된 위험도만 배지로 표시한다.
- 종목 상세 상단 핵심 정보 영역에 지정된 위험도만 배지로 표시한다.
- 상품 설명에는 위험도가 관리자의 정성적·상대적 분류이며 원금 보장이나 무손실을 의미하지 않는다는 문구를 추가한다.

## 검증

- 미지정/저위험/고위험 값이 포트폴리오 정규화 과정에서 보존되는지 단위 테스트한다.
- lint, TypeScript 검사, 전체 단위 테스트를 실행한다.
- Prisma 클라이언트를 재생성하고 운영 DB 스키마에 nullable 컬럼을 반영한 뒤 기존 행이 모두 미지정인지 확인한다.
